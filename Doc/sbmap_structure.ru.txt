Структура файла уровней
~~~~~~~~~~~~~~~~~~~~~~~

I. Введение
  1. Файл уровней является текстовым файлом в кодировке ASCII и является набором из блоков информации.
  2. Файл имеет расширение ".sbmap".
  3. Файл начинается с строки "sokoban_game_maps:$$", где "$$" -- двузначный номер версии файла.
  4. Каждый уровень является отдельным блоком.
  5. Раздел блоков начинается пустой строкой, и каждый блок отделятеся так же пустой строуой, кроме последнего, т.к. он в конце файла.
  6. В начале строки двойной слэш "//" показывает, что строка должна игнорироваться загрузчиком уровней.
     Комментарии могут находится в любом месте файла: как внутри блока, так и отдельными строками.
     Комментарии, принадлежащие блоку уровня должны начинатся на следующей строке после последней строки карты.
     Комментарии, принадлежащие ВСЕМУ файлу карты, должны отделятся пустыми строками.


II. Специальные знаки карты уровня
  1. ' ' -- Floor
  2. '#' -- Wall
  3. '-' -- Stock
  4. '.' -- None
  5. '+' -- Box
  6. '@' -- Player

  Пример:
  ~~~~~~
    ..###...
    ..#-#...
    ..#+####
    ###  +-#
    #-+ @###
    ####+#..
    ...#-#..
    ...###..
    // Комментарий
    // блока 
    // этого уровня


III. Структура блока уровня
  1. Каждый уровень представляет собой отдельный блок.
  2. 1-я строка уровня -- "level:$l", где $l -- номер уровня; уровни начинаются с единицы.
  3. 2-я строка уровня -- "size:$W*$H", где $W - ширина, $H - высота.
     При этом уровень не может быть менее чем 8 по высоте и 6 по ширине.
  4. начиная с 3-й строки и по H+3 -- идут строки уровня.

  ***. Если блоки уровней с одним и тем же номером повторяются, то берётся последний.
